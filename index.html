Projects


Fluid Dynamics Particle Simulator


  • This is a realistic CGI water simulator that utilizes C and Blender. The water physics are derived from a multitude of fluid dynamics research papers in order to mimic how water moves in real life.

  • Through multi-threading and hashing, calculating the interaction of the tens of thousands of particles under the rules of fluid dynamics, as well as graphics rendering, via the marching cubes algorithm, was fast.


Cryptographic File Sharing System


  • This is a multi-user, safe and secure file sharing and storage system that utilizes many cryptographic tools including public and private key encryption, HMAC, AES-CTR block chaining, SHA-512 hashing, and symmetric encryption.

  • Through the utilization of a transient data design philosophy, exlcuding a key and data store, the confidentiality, integrity, and authenticity of user data and files.


Pintos Operating System


  • This is an operating system that I designed and implemented various enhancements on, including user program execution, system calls, and thread management.

  • Two specific things I implemented were: a priority donation algorithm for multi-thread synchronization functionality to allow for multiple threads to simaltaneously use locks, semaphores, and condition variables for atomic actions. As well as a least recently used buffer cache for an i-node based file system for a faster runtime speed.


Gitlet


  • This is a fully functioning local file version control system that mimics some of the main functions of Git including branching, commiting, checking out, logging, resetting, and merging.

  • This was the first project I did where I was really challenged and this led me to cementing the importance of a well thought out design and good coding habits. It is thanks to this project that I am where I am today.


RookieDB


  • This is a database where I designed and developed many diverse functionalities for, including B+ tree indicies for faster database storage access, efficient join algorithms, query optimizations, a complex multi-granularity locking system for the concurrent execution of transactions, and database recovery through logging.

  • For the multi-granularity locking system, I utilized many different lock types on all parts of the database in order to allow for atomic actions and to maintain database consistency, as well as a wait queue for multiple transactions requesting access to currently locked resources.


Go Motivation


  • This is a full stack web app that helps motivate users to accomplish their long term fitness goals through a calendar and achievement system. For API's, we used a calendar API, and information storage API for keeping track of progress.

  • My team and I utilized the full user interface design development process, including A/B testing, persona crafting, and wireframing, in order to build the app from the ground up for iterative improvements and in order to cater towards a specific focus group.


Integer Linear Program Room Optimizer

  • This is a NP-complete search problem where my team and I had to find the most optimal sets of people and rooms where certain conditions were met, or in other words, generate the most optimal set of vertex groupings that maximize the sum total edge values, while adhering to an upper bound threshold for the sum total graph traversal edge values.

  • We utilized integer linear programming to find the optimal set of vertex groupings given 100 or fewer vertices. And placed 10th out of 243 groups in a contest that tested the number of inputs our solver could process given a time limit.
